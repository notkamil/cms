# Техническое задание: система управления коворкингом

## 1. Назначение и цели

Система предназначена для учёта пространств коворкинга, бронирований, подписок и платежей. Цели: предоставить участникам возможность просматривать доступные пространства и бронировать их (разово или по подписке), администрации — управлять тарифами, пространствами, персоналом и контролировать финансы.

## 2. Пользователи

- **Участник (Member)** — клиент коворкинга. Регистрируется в системе, пополняет баланс, оформляет подписки или разовые бронирования, привязывает участников к бронированию.
- **Администратор** — сотрудник коворкинга. Ведёт справочники (типы пространств, удобства, тарифы, пространства, персонал), контролирует бронирования и подписки.

## 3. Основные функции

1. **Справочники** — типы пространств, удобства, тарифы (почасовые, месячные, пакеты), перечень пространств с привязкой к типам и удобствам, персонал.
2. **Участники** — регистрация, профиль, просмотр баланса.
3. **Работа с финансами** — пополнение и списание средств (при оплате бронирований и подписок), прочие операции по балансу; история операций по балансу для участника (в личном кабинете).
4. **Подписки** — оформление подписки на тариф с датами и остатком времени (в минутах); учёт активных/истёкших/отменённых; для тарифов с фиксированным пространством — автоматическое создание бронирования на период подписки.
5. **Бронирования** — разовое (оплата по тарифу с баланса) и по подписке (списание остатка минут); привязка участников к бронированию; контроль пересечений по пространству и времени.
6. **Пространства** — статусы (доступно / занято / обслуживание / отключено); отображение свободных слотов на дату/время.
